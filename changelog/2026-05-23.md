### やったこと

#### Webカメラタイムラプス機能を追加（タイマー連動）
- 仕様
  - タイマー開始/停止に連動して Web カメラ映像を一定間隔でキャプチャし、MP4 に変換して保存
  - 保存先：`%APPDATA%\com.minimaltimer.app\timelapses\`
  - 設定：連動 ON/OFF、カメラ選択、撮影間隔（秒）、再生 FPS、解像度（640×480 / 1280×720 / 1920×1080）
  - 設定 UI は記録ボタンの右クリックメニューに「タイムラプス設定」を追加
  - 録画中は円の左上に赤い「●REC」インジケーターを表示
- 実装方針
  - WebCodecs `VideoEncoder`（H.264 baseline / `avc1.42E01F`）でフレームをエンコード
  - `mp4-muxer` ライブラリ（npm 経由でインストール、ESM を `src/vendor/` に vendor 化）で MP4 コンテナを生成
  - 撮影時刻ではなく 1/FPS 間隔の論理タイムスタンプを付与することで、撮影 10s 間隔 → 再生 30FPS のような真のタイムラプス圧縮を実現
- 変更ファイル
  - 新規：[src/timelapse.js](src/timelapse.js)（録画・エンコード・保存の制御）
  - 新規：[src/vendor/mp4-muxer.mjs](src/vendor/mp4-muxer.mjs)（vendor 化）
  - 変更：[src/index.html](src/index.html)（設定パネル、コンテキストメニュー項目、REC インジケーター追加）
  - 変更：[src/style.css](src/style.css)（タイムラプス UI のスタイル）
  - 変更：[src/main.js](src/main.js)（`st.running` を `Object.defineProperty` でラップし、true/false の遷移で `timelapse.start()` / `timelapse.stop()` を呼ぶ）
  - 変更：[src-tauri/src/lib.rs](src-tauri/src/lib.rs)（`save_timelapse_mp4` と `open_timelapses_folder` コマンド追加）
  - 変更：[package.json](package.json)（`mp4-muxer` 依存追加）
- 競合対策

# 記録機能強化 計画書

作成日：2026-03-04

---

## 1. 現状分析

### 現在できていること

| 項目 | 状態 |
|------|------|
| セッション保存 | `{task, duration, timestamp}` を localStorage に保存 |
| 記録ページ表示 | タスク別合計時間をバーグラフで表示（今日 / 全期間） |
| リセット | 全ログを一括削除 |

### 現状の問題点・足りないもの

1. **開始時刻が記録されていない**
   `timestamp` は保存時刻（ほぼ終了時刻）のみ。「何時から何時まで動かしたか」が分からない。

2. **セッション単位の一覧が見られない**
   タスク別に集計した合計値しか表示できず、個々のセッション履歴を確認できない。

3. **日付フィルタが粗い**
   「今日」「全期間」の2択しかない。週・月単位で振り返れない。

4. **記録のエクスポートができない**
   データが localStorage に閉じており、スプレッドシート等への持ち出しが困難。

---

## 2. 目標

「**いつ・何を・どのくらいやったか**」を時系列で確認できるようにする。

---

## 3. データ設計

### 3-1. 保存するログの構造（変更）

現在：
```json
{ "task": "タスク名", "duration": 1500, "timestamp": 1709500000000 }
```

変更後：
```json
{
  "id": "uuid-or-random-id",
  "task": "タスク名",
  "duration": 1500,
  "startedAt": 1709498500000,
  "endedAt":   1709500000000,
  "mode": "countdown"
}
```

| フィールド | 型 | 説明 |
|------------|-----|------|
| `id` | string | セッション識別子（ランダム生成） |
| `task` | string | タスク名（未設定時は空文字） |
| `duration` | number | 実際に動かした秒数 |
| `startedAt` | number | タイマーを最後に開始したUNIXミリ秒 |
| `endedAt` | number | タイマーが止まったUNIXミリ秒 |
| `mode` | string | `"countdown"` または `"countup"` |

**後方互換性**: 旧フォーマット（`timestamp`のみ）のログも読めるよう、`endedAt ?? timestamp` でフォールバック。

## 4. アクセス経路

既存の `btn-records`（メイン画面ホバー時に表示される時計アイコンボタン）をクリックすると、別ウィンドウとして `records.html` が開く。この**既存の記録ウィンドウ内**に「集計ビュー」と「一覧ビュー」の切替を追加する。エントリポイント（ボタンの位置・動作）は変えない。

---

## 5. 機能要件

### 5-1. タイマー側（main.js）の変更

- タイマー開始時に `st.sessionStart = Date.now()` を記録する
- `logSession()` の保存データに `startedAt`・`endedAt`・`mode` を追加する
- 5秒未満の除外ルールは維持

### 5-2. 記録ページ（records.html / records.js）の変更

#### タブ構成の拡張

| タブ | 内容 |
|------|------|
| 今日 | 現状維持（タスク別合計） |
| 今週 | 今週月曜〜本日の集計（新規） |
| 今月 | 当月の集計（新規） |
| 全期間 | 現状維持 |

#### 表示ビューの追加

**① 集計ビュー（現状維持・改善）**
- タスク別バーグラフ
- 総時間をフッターに表示

**② セッション一覧ビュー（新規）**
- セッションを日付グループで表示
- 各行：`開始時刻 → 終了時刻 ／ タスク名 ／ 時間`
- 例：`10:00 → 10:25 ／ 読書 ／ 25m`

```
─ 2026-03-04（今日）─────────────────────
  10:00 → 10:25   読書          25m
  14:00 → 14:50   コーディング  50m

─ 2026-03-03（昨日）─────────────────────
  09:15 → 10:00   コーディング  45m
```

#### エクスポート機能（新規）

- CSV ダウンロードボタンを追加
- 出力列：`日付, 開始時刻, 終了時刻, タスク, 秒数`
- Tauri の `fs` API または `<a download>` でファイル保存

## 7. 実装ステップ

| ステップ | 作業内容 | 影響ファイル |
|----------|---------|-------------|
| 1 | `logSession()` を修正し `startedAt`・`endedAt`・`mode` を保存 | `main.js` |
| 2 | タイマー開始時に `st.sessionStart` をセット | `main.js` |
| 3 | `records.js` に旧ログのフォールバック読み込みを追加 | `records.js` |
| 4 | 期間タブを「今日／今週／今月／全期間」に拡張 | `records.js`, `records.html` |
| 5 | セッション一覧ビューを実装（日付グループ表示） | `records.js`, `records.html`, `records.css` |
| 6 | 集計／一覧のビュー切替ボタンを追加 | `records.html`, `records.css` |
| 7 | CSV エクスポート機能を実装 | `records.js` |

---

## 8. 優先度

| 優先度 | 機能 | 理由 |
|--------|------|------|
| 高 | `startedAt`・`endedAt` の記録（ステップ1〜2） | 後から補完不可能な情報 |
| 高 | セッション一覧ビュー（ステップ3〜6） | 「動かした時間を見る」本体 |
| 中 | 週・月タブ（ステップ4） | 集計精度の向上 |
| 低 | CSV エクスポート（ステップ7） | 便利だが後回し可 |

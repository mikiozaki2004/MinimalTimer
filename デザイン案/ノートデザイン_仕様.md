# ノートデザイン 詳細仕様

作成日：2026-07-21 ／ 採用案：`縦長リデザイン_モック20案.html` の 02「ノート」
プロトタイプ：`ノートデザイン_プロトタイプ.html`

---

## 1. コンセプト

**「作業中デスクに開いておく、1 冊のノート」**。
タイマーは日誌のヘッダー、進行中の作業は罫線に手書きした ToDo。
文房具のメタファーで状態を表現し、ミニマルさと親しみを両立する。

| 状態 | 文房具メタファー |
|------|-----------------|
| 走行中（自分/AI が進行中） | **蛍光マーカー**でスッと引いたハイライト |
| 完了 | **赤ペンの見せ消し**（取り消し線） |
| 前セットからの持ち越し | 左マージンに **赤ペンの継続マーク `↪`** |
| ポモドーロのセット進捗 | **タリー（正の字／画線）** |

## 2. カラー（ライト＝クリーム紙）

| トークン | 値 | 用途 |
|----------|-----|------|
| `--paper` | `#FBF6E9` | 紙の地色（ウォームクリーム） |
| `--rule`  | `rgba(90,120,150,.22)` | 横罫線（うすい藍） |
| `--margin`| `rgba(200,70,60,.40)` | 左マージンの赤い縦線 |
| `--ink`   | `#3A362E` | 本文インク（温かい黒） |
| `--pencil`| `#8A8577` | 時間表示（鉛筆グレー） |
| `--redpen`| `#C0392B` | 完了の取り消し線・継続マーク |

## 3. タイポグラフィ

| 用途 | フォント | 補足 |
|------|---------|------|
| 作業テキスト・見出し・追加欄 | **手書き系**（Web: `Yomogi`／JP 手書き。バンドル前提） | ノートらしさの核 |
| タイマー数字 | セリフ（`Georgia` 等）or 手書き | 視認性優先。プロトは Georgia |
| 現在時刻・日付ヘッダー | UI サンセリフ小 | “印刷された日誌の見出し”の質感 |
| 時間（累積） | 手書き or 等幅小・`--pencil` | 走り書きの数字 |

> デスクトップアプリ実装時は Yomogi 等をアプリに**同梱**（オフライン動作のため）。

## 4. 紙面レイアウト

```
┌─────────────────────────┐  幅 ≈ 240px
│ ∿∿∿ リング綴じ ∿∿∿       │  上部の綴じ（装飾）
│ 📌            7/21(月) 14:30 ∨│  ピン / 日付 / 時刻 / 開閉
│                           │
│        24:12              │  タイマー（セリフ大）
│    ‿‿‿‿‿‿‿‿ 62%          │  手描き風アンダーライン＝進捗
│    セット |||／||||        │  タリー（現在 2/4）
│┌─マージン(赤)             │
││  やること ‿‿             │  見出し（手書き＋波線）
││ ☑ 要約をAIに依頼   05:20  │  完了＝赤で見せ消し
││ ☐ [▨LP文章を書く]  12:48 │  走行＝黄マーカー
││↪☐ [▨コード生成]    08:03 │  持ち越し＝赤↪
││ ☐ メール返信       —     │
││ ＋ 書き足す…              │  追加欄
│└─────────────────────────│
│  ▶  ⏸  ⟳    ☕ 🍅 🌗      │  操作（ホバーで出現）
└─────────────────────────┘
```

- 横罫線はリスト領域のみ。各行の高さ＝罫線ピッチ（≈30px）に合わせ、**文字が線に乗る**。
- 左マージンの赤縦線から本文をインデント。継続マーク `↪` はマージン内に置く。

## 5. 各要素の仕様

### 5-1. チェックボックス
- 手描き感のある角丸スクエア（わずかに回転）。
- **完了時**：赤ペンのチェック `✓` が枠に重なり、テキストに**赤い取り消し線**。

### 5-2. 走行中（ハイライト）
- 作業名の背後に蛍光マーカーの帯（`--marker`）。左右端を少し滲ませる。
- マーカーは呼吸するように**ごく淡く明滅**（`opacity` 0.45↔0.6、3s）。過剰にしない。
- 時間は `--pencil`。自分＝そのまま／AI＝末尾に小さく `AI`（任意）。

### 5-3. 完了（見せ消し）
- セット終了の瞬間は取り消し線で数秒残す → フェードでアーカイブ（§4-4）。

### 5-4. 持ち越し（キャリーオーバー）
- 前セットから残った未完了は、マージンに赤 `↪`。
- 追加のツールチップ等は不要（ミニマル維持）。線一本で伝える。

### 5-5. セット・タリー
- ポモドーロのセット数を画線法で表示。完了セット＝濃いインク、これから＝うすい鉛筆。
- 例）4 セットで 2 本目 → `❙❙|  |`（2 本濃く、残りうすく）。

### 5-6. 進捗アンダーライン
- タイマー直下に手描き風の下線。左から `--accent` で 62% 塗り、残りは鉛筆のかすれ。

### 5-7. 親作業・子作業（グループ）★
- **親**：見出し級の手書き（やや大きめ）＋ 左マージンの三角 `▾/▸` で子を開閉。
  右端に **集計**（`完了子数/子数 · 子の合計時間`）を鉛筆書きで表示。
- **子**：親の下にインデントし、**手描き風のブラケット罫**（`│`＋短い `─`）で連結。
  子の行はそのままチェック／走行マーカー／時間を持つ（末端が計測単位）。
- **チェック連動**：親のチェック＝全子を完了（赤ペン）。全子完了で親も自動的に見せ消し。
- **走行の集約**：子のどれかが走行中なら、親の見出しにも**淡いマーカー**を敷いて「このグループが動いている」を示す。
- **持ち越し**：未完了の子が残る親は、`↪` 付きでグループごと次セットへ（§4-4／親名の前に赤 `↪`）。

```
▾ ☐ LP制作                 1/3 · 26:11   ← 親（集計）
   │─ ☑ 要約をAIに依頼        05:20        ← 子（完了・赤見せ消し）
   │─ ☐ [▨文章を書く]         12:48        ← 子（走行・黄マーカー）
   │─ ☐ [▨コード生成(AI)]     08:03        ← 子（走行）
  ☐ メール返信                —            ← 単独タスク（子なし）
```

## 6. インタラクション

| 操作 | 挙動 |
|------|------|
| タイマー部／`∨` クリック | リスト開閉（既定は畳んで最小占有。§A案の踏襲） |
| チェックボックス | 完了トグル（赤チェック＋見せ消し） |
| ＋ 書き足す | インライン入力 → Enter で行追加 |
| 行の ▸ | その作業の計測開始／停止（並列可） |
| 🌗 | ライト（紙）⇔ ダーク（黒板）切替 |
| ホバー | 下部の操作アイコンがフェードイン（既存のステルス演出を踏襲） |

## 7. 実装メモ（既存資産の活かし方）

- CSS 変数（`--paper` `--ink` `--accent` …）でライト/黒板を切替 → 既存のテーマ機構（`body.light` 等）に無理なく接続。
- 罫線・マージンは擬似要素＋`repeating-linear-gradient` で画像不要・軽量。
- マーカー／見せ消しは追加 DOM 無しでクラス（`.run` `.done` `.carried`）だけで表現でき、
  §4 のデータモデル（`running`/`done`/`bornSet`）と 1:1 で対応する。
- 手書きフォントのみ**同梱アセット**が増える点に注意（それ以外は現状の軽量方針を維持）。
